## 1. 系统概述

智能健身房管理系统是一个全栈 Web 应用，提供健身房日常运营管理功能，包括：

- **会员管理** — 会员信息的增删改查
- **教练管理** — 教练信息管理
- **课程管理** — 课程信息管理
- **课程预定** — 在线查看排期并预定课程
- **器材管理** — 器材信息管理
- **器材监控** — 模拟实时监控器材使用状态
- **数据看板** — 运营数据统计图表展示

## 2. 环境要求

在运行本系统前，请确保电脑已安装以下软件：

| 软件 | 版本要求 | 用途 |
|------|----------|------|
| JDK | 17 或更高 | 运行后端 Java 程序 |
| MySQL | 8.0 或更高 | 数据存储 |
| Node.js | 18 或更高 | 运行前端 |
| Maven | 3.8 或更高 | 后端构建工具 |

> **Windows 用户也可以使用 IDEA（IntelliJ IDEA）** 一站式管理后端，它会自带 Maven。

## 3. 环境安装指南

### 3.1 安装 JDK 17

1. 访问 [Oracle JDK 下载页](https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html) 或 [Adoptium](https://adoptium.net/)
2. 下载 Windows x64 安装包，运行安装
3. 安装完成后打开命令行验证：
   ```bash
   java -version
   ```
   出现 `java version "17.x.x"` 即成功

### 3.2 安装 MySQL 8.0

**方案一：MySQL Installer（推荐新手）**

1. 访问 [MySQL 下载页](https://dev.mysql.com/downloads/installer/)
2. 下载 `mysql-installer-community-8.0.x.msi`
3. 安装时选择 **Server only**，一路下一步
4. 设置 root 用户密码（建议设为 `123456`，方便后续配置）
5. 安装完成后，在 Windows 服务中确认 MySQL 服务已启动

**方案二：使用集成环境（如 PhpStudy / 小皮面板）**

1. 下载安装 [小皮面板](https://www.xp.cn/)
2. 启动 MySQL 5.7 或 8.0
3. 默认端口 3306，默认用户 root，密码根据设置填写

验证安装：
```bash
mysql -u root -p
```
输入密码后进入 MySQL 命令行即成功。

### 3.3 安装 Node.js

1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载 LTS 版本（18.x 或 20.x）
3. 运行安装包，一路下一步
4. 验证安装：
   ```bash
   node -v
   npm -v
   ```

### 3.4 安装 Maven

1. 访问 [Maven 下载页](https://maven.apache.org/download.cgi)
2. 下载 `apache-maven-3.9.x-bin.zip`
3. 解压到 `C:\Program Files\apache-maven-3.9.x`
4. 配置环境变量：
   - 右键"此电脑" → 属性 → 高级系统设置 → 环境变量
   - 新建系统变量 `MAVEN_HOME`，值填解压路径
   - 编辑 Path 变量，添加 `%MAVEN_HOME%\bin`
5. 验证：
   ```bash
   mvn -v
   ```

## 4. 数据库配置

### 4.1 创建数据库

打开命令行，执行：

```bash
mysql -u root -p
# 输入密码后进入 MySQL

# 创建数据库
CREATE DATABASE gym_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 退出
exit
```

### 4.2 初始化表结构（可选）

系统启动时会自动执行初始化脚本，一般不需要手动操作。如果需要手动导入，可以执行：

```bash
mysql -u root -p gym_db < schema.sql
mysql -u root -p gym_db < seed-data.sql
```

> `schema.sql` 在项目根目录，包含所有表结构。  
> `seed-data.sql` 包含初始化账号和演示数据。

### 4.3 修改数据库连接配置

打开 `backend/src/main/resources/application.yml`，找到数据库配置部分：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/gym_db?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai&characterEncoding=utf8
    username: root           # 改成你的 MySQL 用户名
    password: 123456         # 改成你的 MySQL 密码
```

> 如果 MySQL 密码不是 123456，请务必修改 `password` 字段。

## 5. 启动后端

### 方式一：双击启动脚本（推荐）

找到 `backend/start-backend.bat`，双击运行。

脚本会自动检测可用端口（默认从 18091 开始），输入 `Y` 确认后启动后端。

### 方式二：命令行启动

```bash
cd backend
mvn spring-boot:run
```

> 首次运行会自动下载 Maven 依赖，需要联网，耗时 1-5 分钟。

### 方式三：IDEA 启动（推荐开发用）

1. 用 IntelliJ IDEA 打开项目根目录
2. IDEA 会自动识别为 Maven 项目并下载依赖
3. 找到 `GymApplication.java`（路径：`backend/src/main/java/com/example/gym/GymApplication.java`）
4. 点击类旁边的绿色三角 ▶ 运行

### 启动成功的标志

控制台出现类似以下信息：
```
Tomcat started on port 18091 (http) with context path '/'
Started GymApplication in X.XXX seconds
```

## 6. 启动前端

### 方式一：双击启动脚本（推荐）

找到 `frontend/start-frontend.bat`，双击运行。

输入后端端口号（默认 18091，直接回车即可），前端会自动启动。

### 方式二：命令行启动

```bash
cd frontend

# 安装依赖（首次启动需要，只需执行一次）
npm install

# 启动开发服务器
npm run dev
```

> `npm install` 只需执行一次，后续启动直接 `npm run dev` 即可。

### 启动成功的标志

控制台显示：
```
VITE v5.x.x  ready in XXXX ms
➜  Local:   http://localhost:5174/
```

## 7. 访问系统

打开浏览器，访问：**http://localhost:5174**

### 启动顺序（重要）

```
1️⃣ 启动 MySQL 服务
2️⃣ 启动后端（start-backend.bat 或 IDEA）
3️⃣ 启动前端（start-frontend.bat 或 npm run dev）
4️⃣ 浏览器打开 http://localhost:5174
```

> 后端和前端都启动完成后，才能正常登录和使用系统。

## 9. 功能模块说明

### 9.1 登录页面

- 选择角色下拉框（管理员 / 教练 / 会员）
- 输入用户名和密码
- 点击登录

### 9.2 数据看板（Dashboard）

- 登录后默认进入数据看板
- 展示会员总数、教练总数、器材总数、课程总数等统计
- 图表展示运营趋势

### 9.3 会员管理

- 左侧菜单 → 会员管理
- 支持：新增会员、编辑会员信息、查看详情、删除会员
- 列表支持分页展示

### 9.4 教练管理

- 左侧菜单 → 教练管理
- 支持：新增教练、编辑、查看详情、删除

### 9.5 课程管理

- 左侧菜单 → 课程管理
- 支持：新增课程、编辑课程信息、删除课程

### 9.6 课程预定

- 左侧菜单 → 课程预定 → 课程排期
- 查看所有课程排期，点击预定按钮进行预定
- 左侧菜单 → 课程预定 → 我的预定
- 查看自己的预定记录，可取消预定
- 可对已完成的课程进行评价

### 9.7 器材管理

- 左侧菜单 → 器材管理 → 器材列表
- 支持：新增器材、编辑、删除
- 左侧菜单 → 器材管理 → 器材监控
- 模拟实时查看器材使用状态（在线/离线/使用中）

## 10. 常见问题

### Q1：启动后端时报端口被占用

**原因：** 18091 端口已被其他程序占用。

**解决：** `start-backend.bat` 会自动尝试递增端口（18091 → 18092 → 18093 ...）。  
如果启动前端时改了端口，记得在 `start-frontend.bat` 中输入对应的后端端口号。

### Q2：后端启动失败，报数据库连接错误

**原因：** MySQL 服务未启动，或数据库连接配置不对。

**解决：**
1. 确认 MySQL 服务已启动（任务管理器 → 服务 → MySQL80）
2. 确认已创建 `gym_db` 数据库
3. 检查 `application.yml` 中的用户名和密码是否正确

### Q3：前端页面打不开

**原因：** 前端服务器未启动或端口被占用。

**解决：**
1. 确认已执行 `npm install`
2. 查看控制台是否有错误提示
3. 手动指定端口：`npm run dev -- --port 5175`

### Q4：登录提示"用户名或密码错误"

**解决：**
1. 确认选择了正确的角色（管理员 / 教练 / 会员）
2. 默认账号密码见 [第 8 节](#8-默认账号)
3. 如果修改过数据库，确认 `seed-data.sql` 已正确导入

### Q5：Maven 下载依赖太慢

**解决：** 在 `backend/pom.xml` 中添加阿里云镜像（在 `<repositories>` 部分）：

```xml
<repositories>
    <repository>
        <id>aliyun</id>
        <url>https://maven.aliyun.com/repository/public</url>
    </repository>
</repositories>
```

### Q6：npm install 太慢

**解决：** 使用国内镜像：

```bash
npm config set registry https://registry.npmmirror.com
```

### Q7：如何关闭系统？

- 前端：在命令行按 `Ctrl + C`，输入 `Y` 确认
- 后端：在命令行按 `Ctrl + C`
- 双击启动的脚本，直接关掉命令行窗口即可
